gin web框架快速入门

May 18, 2019
Development • golang
go • golang • development • gin

web框架Gin

What is Gin

Gin是一款开源的轻量级web服务框架，基于http-router搭建而成

它提供了类似martini的API实现但同时在性能上具有巨大优势

Why Gin

Golang的web服务框架有很多，诸如beego,iris,gin等等，选用Gin作为例子并不是否认其它框架，一是用多了比较习惯，二是可以快速搭建一个API架子

Install

要安装Gin,只需要国际惯例的go get

go get github.com/gin-gonic/gin

Start

创建一个新项目，请注意，如果需要使用包管理，则默认情况下你不能把项目目录建在GOPATH中

在main.go中加入以下代码

package main

import "github.com/gin-gonic/gin"

func main(){
	router := gin.Default()

	router.GET("/", func(c *gin.Context) {
		c.String(200,"hello world!")
	})

	router.Run(":8888")
}

然后编译运行

(Linux)

go build -o app ./app

(Windows)

go build -o app.exe 运行 app.exe

出现下列提示则代表服务已经启动，如果提示8888端口被占用，那换一个即可

[GIN-debug] [WARNING] Creating an Engine instance with the Logger and Recovery middleware already attached.

[GIN-debug] [WARNING] Running in "debug" mode. Switch to "release" mode in production.
 - using env:	export GIN_MODE=release
 - using code:	gin.SetMode(gin.ReleaseMode)

[GIN-debug] GET    /                         --> main.main.func1 (3 handlers)
[GIN-debug] Listening and serving HTTP on :8888

这个时候打开浏览器，输入http://127.0.0.1:8888/即可看到hello world了

讲解

通过上面的简单例子，我们实现了一个小的web服务器，那么每一行代码具体实现了什么呢

框架模板

首先是gin.Default()

	router := gin.Default()

Default()方法帮助用户快速生成了一个基本服务框架，包括了默认的recove中间件和logger中间件等一系列服务的模板，通过此模板你可以快速生成一个web服务引擎而不需要过多的配置，通常来说，Default()的模板足以满足你大多数情况下的需求

路由注册

	router.GET("/", func(c *gin.Context) {
		c.String(200,"hello world!")
	})

这是一个完整的路由注册模块，可以看到我们通过GET()方法将一个闭包注册到了/路由上(即例子中的http://127.0.0.1:8888/)

我们将其分为两部分介绍

注册函数

首先是外部的路由注册函数GET()，代表了注册某一个路由的GET请求，同理，存在着POST(),DELETE(),PUT(),OPTIONS()等和HTTP请求动词一一对应的方法

当你向web服务发送了一个请求后，gin就会检测对应的路由及请求动词，若不存在对应的handler测会404，若存在，则会调用注册的闭包

	r := gin.Default()
	r.GET("/")
	r.POST("/")
	r.PUT("/")
	r.DELETE("/")
	r.PATCH("/")
	r.OPTIONS("/")
	r.HEAD("/")

闭包

然后来讲闭包，根据参数列表可知，每一个路由可注册0-n个闭包，具体如下

	r := gin.Default()

    //零个闭包
	r.GET("/r1")

    //多个闭包
	r.GET("/r2", func(c *gin.Context) {
		
	}, func(c *gin.Context) {

	}, func(c *gin.Context) {

	})

这些闭包会按注册顺序顺序执行，由于存在中间件(将会在以后文章提及)，整体请求处理流程如下

语言级请求处理 -> 框架请求处理 -> 中间件请求处理 -> 注册闭包请求处理

如果不存在闭包，则中间件处理完成后将会返回默认的200和空payload

*gin.Context - Output`

接下来将闭包入参*gin.Context,这其实是经过框架优化的，集成了响应与请求的结构体，可以通过此结构体获取请求参数，也可以通过此结构体发送响应，可以理解为*gin.Context是对整个”HTTP请求”的一个封装

下例子说明了如何通过*gin.Context输出一个JSON对象

package main

import "github.com/gin-gonic/gin"

func main(){
	r := gin.Default()

	r.GET("/", func(c *gin.Context) {
		c.JSON(200,&gin.H{
			"error":0,
			"msg":"success",
		})
	})

	r.Run(":8888")
}

可见，我们只需要在c.JSON()方法中输入状态码和JSON对象即可，由于golang中大部分结构都能被正确的转化为JSON，所以只需要传入合理的变量即可

在这里我们使用了&gin.H{}这样一个对象，其本质其实也就是一个map[string]interface{},只不过Gin对他重载了一些方法，便于框架调用，我们使用的时候只需要将其视为一个map即可

gin.H的定义：

// H is a shortcut for map[string]interface{}
type H map[string]interface{}

当然，除了JSON，还有c.XML(),c.String(),c.Status()等输出方法可供渲染API(或输出文本，或者干脆无输出)

*gin.Context - Input

当然，*gin.Context只负责输出是不够的，还需要控制输入，下面是几个简单的获取常用请求输入的方法

package main

import (
	"fmt"
	"github.com/gin-gonic/gin"
)

func main(){
	r := gin.Default()

	r.POST("/", func(c *gin.Context) {
		//获取URL query参数
		id := c.Query("id")
		//获取POST表单参数
		password := c.PostForm("password")
		//获取请求头参数
		auth := c.GetHeader("Authorization")
		//获取原http.Request结构体
		raw := c.Request
		fmt.Println(id,password,auth,raw)
	})

	r.Run(":8888")
}

简单API

在获得了输入和输出能力之后，我们就可以利用Gin搭建一个简单的API了

如下方例子，实现了一个最基本的用户注册登录的功能

package main

import (
	"github.com/gin-gonic/gin"
)

//用户结构体
type user struct{
	Name string
	Age string
}

//用户字典
var users  map[string]*user

func main(){
	//初始化用户数组
	users := make(map[string]*user)
	
	//初始化Gin引擎
	r := gin.Default()

	//用户注册方法
	r.POST("/user", func(c *gin.Context) {
		//添加用户
		username := c.PostForm("username")
		users[username] = &user{
			Name:c.PostForm("name"),
			Age:c.PostForm("age"),
		}
		//返回JSON
		c.JSON(200,&gin.H{
			"error":0,
			"msg":"success",
			"data":nil,
		})
	})
	
	r.GET("/user", func(c *gin.Context) {
		//获取用户名
		username := c.Query("username")
		//返回数据
		c.JSON(200,&gin.H{
			"error":0,
			"msg":"success",
			"data":users[username],
		})
	})

	r.Run(":8888")
}

对应的接口如下

POST /user 注册用户

PostForm:

  • username 用户名
  • name 姓名
  • age 年龄
GET /user 获取用户信息

Query:

  • username 用户名
